Atividade Prática 2 - Simulação de Entrevista 
1. Roteiro utilizado com anotações 
Pergunta Resposta / anotação

Qual é sua principal função na secretaria?
Realizar matrículas, atualizar  cadastros e emitir documentos. 
Quantos atendimentos ocorrem por dia em períodos de 
matrícula? 
Cerca de 50 ou mais atendimentos  por dia. 

O processo é totalmente digital? 
Não. Parte ainda depende de  conferência e preenchimento  manual. 

Como ocorre a matrícula atualmente? 
Recebimento de documentos,  conferência, cadastro/atualização e  efetivação. 

Quais etapas geram mais problemas? 
Conferência de documentos e  digitação de dados.

A emissão de documentos sai no mesmo dia? 
Na maioria das vezes sim, mas  atrasos acontecem quando há erro  ou lentidão. 

O que você gostaria que o novo sistema melhorasse?
Menos retrabalho, mais rapidez e  menos erros. 

Quais funções seriam mais úteis? 
Busca rápida, cadastro  centralizado, emissão automática e  acompanhamento de status.

3. Lista de requisitos identificados 
RF - Requisitos Funcionais 
 O sistema deve cadastrar e atualizar dados dos alunos. 
 O sistema deve realizar matrículas e consultar informações do aluno. 
 O sistema deve emitir declaração de matrícula e histórico escolar. 
 O sistema deve gerar documentos automaticamente com dados já cadastrados. 
 O sistema deve acompanhar o status da matrícula e das solicitações. 
 O sistema deve exibir alertas de pendências e documentos faltantes.

RNF - Requisitos Não Funcionais 
 O sistema deve ser rápido, principalmente em períodos de matrícula. 
 O sistema deve ser simples e fácil de usar. 
 O sistema deve garantir segurança e confiabilidade dos dados. 
 O sistema deve ficar disponível durante o horário de atendimento.

RN - Regras de Negócio 
 A matrícula só pode ser concluída com dados obrigatórios preenchidos. 
 A emissão de documentos depende de cadastro válido do aluno. 
 Pendências devem ser verificadas antes da conclusão da matrícula. 
 Somente usuários autorizados podem emitir documentos oficiais.
